# Novelty-tail manifests

This directory contains small, explicit QF_UF campaign manifests. The source
of truth is the full hashed corpus manifest generated by
`scripts/bench/fetch_smtlib_qf_uf.sh`; subset manifests must be derived with
`scripts/bench/build_novelty_tail_manifest.py` rather than assembled by hand.

The existing `peq012-size6.jsonl` and `goel-frogs2.jsonl` files are independent
one-source artifacts. The builder has no implicit output path and does not
modify either file.

## Checked 22-source deficit

`shared-z3-yices-deficit-22` is encoded in the builder as an ordered, checked
selection. It is the current 60-second shared Z3/Yices2 coverage deficit:

- 9 Goel sources: 6 SAT and 3 UNSAT;
- 1 UNSAT `PEQ012_size6` source;
- 12 UNSAT QG `qg7` sources.

Each named entry binds its relative path, status, byte count, and SHA-256. A
same-name source or manifest change therefore fails instead of silently
redefining the selection.

Create a portable manifest whose `path` field equals `relative_path`:

```bash
python3 scripts/bench/build_novelty_tail_manifest.py \
  benchmarks/smtlib-2025/qf_uf_manifest.jsonl \
  --selection shared-z3-yices-deficit-22 \
  --source-root benchmarks/smtlib-2025/QF_UF \
  --out benchmarks/novelty-tail/shared-z3-yices-deficit-22.jsonl \
  --report-out benchmarks/novelty-tail/shared-z3-yices-deficit-22.selection.json
```

Portable manifests are resolved by consumers with an explicit corpus root:
`CORPUS_ROOT/QF_UF/...`.

For a manifest pinned to a target machine, add an absolute rebase root. Giving
`--rebase-root` selects rebased mode automatically:

```bash
python3 scripts/bench/build_novelty_tail_manifest.py \
  benchmarks/smtlib-2025/qf_uf_manifest.jsonl \
  --selection shared-z3-yices-deficit-22 \
  --source-root benchmarks/smtlib-2025/QF_UF \
  --rebase-root /work/euf-viper/corpus/QF_UF \
  --out /tmp/shared-deficit.rebased.jsonl \
  --report-out /tmp/shared-deficit.rebased.selection.json
```

Here the output `path` is
`/work/euf-viper/corpus/QF_UF/<relative_path>`; the rebase root need not be the
local verification root.

## Explicit selections

Repeat `--relative-path` in the required output order:

```bash
python3 scripts/bench/build_novelty_tail_manifest.py \
  benchmarks/smtlib-2025/qf_uf_manifest.jsonl \
  --source-root benchmarks/smtlib-2025/QF_UF \
  --relative-path QF_UF/PEQ/PEQ012_size6.smt2 \
  --relative-path QF_UF/2018-Goel-hwbench/QF_UF_frogs.2.prop1_ab_br_max.smt2 \
  --out /tmp/ordered-tail.jsonl \
  --report-out /tmp/ordered-tail.selection.json
```

`--paths-file FILE` is the alternative for a newline-delimited ordered list.
Blank lines, comments, duplicate paths, and combining selection mechanisms are
rejected.

## Validation contract

Before replacing either output, the builder:

1. strictly parses every JSONL row, rejecting blank records, duplicate JSON
   keys, non-finite numbers, missing hash fields, duplicate IDs/paths, and
   malformed or non-QF_UF paths;
2. requires every requested path to occur exactly once and preserves the
   request order;
3. resolves each selected source through `--source-root` or its declared
   manifest path, with containment checks for source-root resolution;
4. verifies the source byte count and SHA-256 and lexically scans the complete
   SMT-LIB file for exactly one top-level `(set-logic QF_UF)` and one top-level
   `(set-info :status sat|unsat)` matching the manifest;
5. emits compact, key-sorted ASCII JSON with no timestamp or host-dependent
   resolved source path; and
6. stages and fsyncs both files before atomically replacing each destination.

The selection report records the input and output manifest hashes, selection
definition hash, ordered source identities, counts, path rewrite mode, and
verification policy. Validation errors return exit status `2` and leave
existing output files unchanged.
